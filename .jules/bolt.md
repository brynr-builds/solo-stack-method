## 2025-06-28 - Component State Isolation for Input Fields
**Learning:** In React, typing in a controlled text input triggers state updates on every keystroke. If this state is managed high up in a component tree (like `PulseBoard.tsx` managing the newsletter email input alongside a large grid of items), it causes the entire component and all its children to re-render constantly while the user is typing, leading to input lag and wasted CPU cycles.
**Action:** Always isolate high-frequency state updates (like text inputs) into their own leaf components whenever possible. In `PulseBoard`, extracting the newsletter form into a separate `PulseNewsletter` component ensures that only the newsletter UI re-renders when the user types their email, leaving the large grid and filter tabs untouched. Also, use `useMemo` for derived data like `filtered` items when the list depends on state that might change (though separating the input state often mitigates the immediate performance hit of re-rendering derived data, memoizing large list filtering is good practice).
