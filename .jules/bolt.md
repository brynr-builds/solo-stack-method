## 2024-06-29 - React Server Cache Optimization for Markdown parsing
**Learning:** In Next.js App Router, using `fs.readFileSync` and `marked.parse` for rendering markdown content is expensive and redundant when called multiple times per request cycle (e.g. for generating static params, metadatas, sitemaps, and page contents). While a simple module-level `Map` works for caching, `cache` from `react` correctly hooks into Next.js's request-scoped memoization (and build-time caching), reducing file system hits safely. However, standard Node execution doesn't support React `cache` (it's a server component construct), so it can't be run by simple `node` or `tsx` scripts without throwing `TypeError: (0 , import_react.cache) is not a function`.
**Action:** Use `import { cache } from 'react'` in Next.js Server environments for expensive data fetching/parsing operations.
